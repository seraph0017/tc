## 页 12/32 [DIAGRAM]

# AI写代码 = 建房子六步流程

```
🏗️ 了解业主需求     → ① 分析需求
   用户到底要什么？

📐 看图纸           → ② 设计架构
   代码分几个文件？怎么配合？

👷 选工人           → ③ 选择技能
   用什么语言？加载哪个Skill？

🔨 施工             → ④ 生成代码
   按SOP一步步写

🔍 验收             → ⑤ 自动验证
   能运行吗？有Bug吗？

🔧 修补             → ⑥ 迭代优化
   有问题就修，直到合格
```

<!-- P13 -->
## 页 13/32 [DIAGRAM]

# 真实案例：AI做"用户管理系统"

```
用户说："做一个用户管理系统
        能注册、登录、管理信息"

① 分析需求：
   注册+登录+查看修改+安全性

② 设计架构（像建筑师画图纸）：
   main.py      ← "大门"
   models.py    ← "户型图"
   auth.py      ← "门锁系统"
   crud.py      ← "物业服务"
   routers/     ← "服务窗口"

③ 加载coding-agent Skill

④ 从底层到上层写代码
   地基→砌墙→装修→通水通电

⑤ 语法+逻辑+安全检查

⑥ 有问题修复后重新验证
```

## 页 19/32 [DIAGRAM]

# 让AI更好用的三个约束原则

```
原则1：明确角色——"你是谁"
  ❌ "帮我写邮件"
  ✅ "你是HR经理，写面试邀请邮件"

原则2：定义标准——"做到什么程度"
  ❌ "帮我写个好的"
  ✅ "段落≤5行，通俗易懂，每个论点配例子"

原则3：提供示例——"像这样做"
  ❌ "帮我写客服回复"
  ✅ "参考范例：'尊敬的XX，感谢反馈…'"
```

---

```

```
有些人特别会给下属布置任务——
