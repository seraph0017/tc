## 页 8/34 [ANALOGY]

# 📋 配置管理 = 规章制度

```
好的物业公司有一本"小区规章制度"：
  保洁排班、安保规则、维修标准、紧急流程
  → 所有规矩写在一个地方
  → 新来的人翻一本就全知道

如果规矩散落在不同人脑子里：
  张师傅记得保洁时间
  李师傅记得安保规则
  → 张师傅请假那天 → 保洁就乱了

AI系统的配置管理：
  → 所有设置集中在一个配置文件里
  → 改一个地方就生效
```

## 页 9/34 [CODE-BILINGUAL]

# OpenClaw的"规章制度总册"

| 配置项 | 白话翻译 |
|--------|---------|
| `environment: production` | 正式运行环境 |
| `memory.update_interval: 14400` | 每4小时更新记忆 |
| `memory.retention_days: 90` | 保留90天记忆 |
| `models.default: kimi-k2.5` | 默认用Kimi |
| `models.fallback: gpt-4o` | Kimi挂了用GPT-4o |
| `agents.sync_interval: 21600` | 每6小时同步记忆 |
| `trading.max_daily_loss: 100` | 日最大亏损$100 |
| `notifications.default: feishu` | 默认飞书通知 |

> 改交易限额？改一个数字
> 换通知渠道？改一个词
> **改一处生效，不用改8个Agent**

## 页 17/34 [QUOTE]

# 📌 教训一：记忆是核心

> **没有记忆的系统只是聊天机器人**
>
> 向量搜索比关键词强大10倍
> 自动收敛比手动整理可靠
> 记忆需要分层
>
> 🏢 **迁移价值：**
> 不记录客户信息、不积累业务知识的公司
> 只是在"一天天重复"
> 而不是"一天天成长"

---

<!-- P18 -->
## 页 18/34 [QUOTE]

# 📌 教训二：分层是必要的

> **专业Agent比通用Agent更有效**
>
> 但协调成本需要控制
> 架构设计决定了上限
>
> 🏢 **迁移价值：**
> 创业初期一人多职能行
> 公司做大了必须分部门
> 关键：找到"分工的合适粒度"

---

<!-- P19 -->
## 页 19/34 [QUOTE]

# 📌 教训三：约束出质量

> **Skill系统提供了必要的结构**
>
> 无约束的输出容易失控
> 麦当劳的秘密是SOP不是厨师
> 测试驱动是必须的
>
> 🏢 **迁移价值：**
> 有标准化流程的团队
> 长期表现一定好于"全凭感觉"的团队

---

<!-- P20 -->
## 页 20/34 [QUOTE]

# 📌 教训四：必须有熔断

> **任何自动化系统都需要熔断机制**
>
> 监控比策略更重要
> 过去不等于未来
> 不是"最好有"是"必须有"
>
> 🏢 **迁移价值：**
> 任何业务都应该有"应急预案"
> "如果出了问题怎么办"
> 应该**事前想好**，不是事后抓瞎

# 🎓 AI时代四条素养

```
素养一：能判断AI适不适合
  → 任务有明确输入输出吗？
  → 允不允许出错？代价多大？

素养二：能提出靠谱的AI需求
  → 角色、输入输出、安全边界、监控指标

素养三：能识别AI产品的"虚"与"实"
  → "AI驱动"≠ 全自动
  → "99%准确"≠ 不出错

素养四：能参与AI相关决策
  → 用记忆/分层/约束/信任公式思维
```
