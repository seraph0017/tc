### 模块 5.2 -- 数据获取与特征工程

# BinanceDataFetcher 设计策略

**先查本地缓存，不足时才请求API**

三大好处：
1. 减少API调用 → 避免限流封IP
2. 历史数据不变 → 无需重复下载
3. 回测速度 → 本地读取快10倍

```
       |

  查本地SQLite缓存
       |

  ├── 是 --> 直接返回缓存数据

```

> 90%阈值：避免因缺少最新1-2条而触发全量请求

### 缓存策略核心代码

```python
def fetch_ohlcv(self, symbol: str = 'ETH/USDT',
                timeframe: str = '4h',
                limit: int = 500) -> pd.DataFrame:
    # 先检查本地缓存
    cached_data = self._get_cached_data(symbol, timeframe, limit)
    if len(cached_data) >= limit * 0.9:  # <-- 重点
        return cached_data

    # 从交易所获取增量数据
    since = None
    if len(cached_data) > 0:
        since = int(cached_data['timestamp'].max()) + 1  # <-- 重点

    ohlcv = self.exchange.fetch_ohlcv(
        symbol, timeframe, since=since, limit=limit
    )
```

## [CONCEPT] Slide 5-5g

### 风险管理五大通用原则

1. **先建防线再建功能**
   -- 自动化之前先设计"停止条件"

2. **多重检查，层层设防**
   -- 不依赖单一检查点

3. **熔断不可逆（需手动恢复）**
   -- 防止系统在异常中反复尝试

4. **监控比策略更重要**
   -- 策略可以赔钱，但你必须知道在赔钱

5. **量化风险用分数（0-100）**
   -- 便于设置自动化阈值

## [SUMMARY] Slide 5-6a

### 第5课核心总结

1. **四层架构** -- 数据→策略→执行→监控，每层独立可替换
2. **特征工程** -- 从OHLCV中提取12个技术指标，为ML提供输入
3. **ML策略** -- 随机森林投票 + 置信度≥0.6才执行
4. **网格策略** -- 不预测方向，等比网格利用波动赚差价
5. **风险管理（核心）** -- 六重安全检查 + 熔断机制 + 风险评分(0-100)

> **最重要的一点：风控原则适用于任何AI自动化系统，不只是交易**
